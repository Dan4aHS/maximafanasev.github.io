Числа

  1. В каком формате хранятся числа?
    2. JavaScript хранятся в 64-битном формате IEEE-754 "числа с плавающей точкой двойной точности"
  2. Для чего нужен Bigint
    3. когда необходимо работать со значениями более чем (2**53-1) или менее чем -(2**53-1)
  3. Как укороченно записать большое/маленькое число
    4. let billion = 1e9; 1000000000
let ms = 1e-6; 0.000001;
  4. шестнадцатеричные (для чего используются?)
    5. для представления цветов, кодировки символов и многого другого
alert( 0xff ); // 255
alert( 0xFF ); // 255 (то же самое, регистр не имеет значения)
  5. двоичные, восьмеричные
    6. let a = 0b11111111; // бинарная форма записи числа 255
let b = 0o377; // восьмеричная форма записи числа 255

alert( a == b ); // true, с двух сторон число 255
  6. строковое представление в системе base
    7. Метод num.toString(base) возвращает строковое представление числа num в системе счисления base
let num = 255;

alert( num.toString(16) );  // ff
alert( num.toString(2) );   // 11111111
  7. Округление В большую В меньшую до ближайшего С удалением дробной части 
    8. Math.floor 3.1 становится 3, а -1.1 — -2
Math.ceil 3.1 становится 4, а -1.1 — -1
Math.round 3.1 становится 3, 3.6 — 4, а -1.1 — -1
Math.trunc 3.1 становится 3, а -1.1 — -1
  8. округляет число до n знаков после запятой и возвращает строковое представление результата
    9. toFixed(n)
  9. Проверка на isFinite isNaN
    10. 1) Иногда isFinite используется для проверки, содержится ли в строке число
let num = +prompt("Enter a number", '');

// вернёт true всегда, кроме ситуаций, когда аргумент - Infinity/-Infinity или не число
alert( isFinite(num) );

2) alert( isNaN(NaN) ); // true
3) alert( isNaN("str") ); // true
4) alert( NaN === NaN ); // false
  10. parseInt и parseFloat
    11. 1) Они «читают» число из строки. Если в процессе чтения возникает ошибка, они возвращают полученное до ошибки число. Функция parseInt возвращает целое число, а parseFloat возвращает число с плавающей точкой:

2) parseInt(str, radix)
alert( parseInt('0xff', 16) ); // 255
alert( parseInt('ff', 16) ); // 255, без 0x тоже работает

alert( parseInt('2n9c', 36) ); // 123456
